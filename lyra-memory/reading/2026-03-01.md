## Key Findings

### Functional Design Patterns Shift
The functional programming paradigm is replacing traditional OOP design patterns. Rather than building via inheritance and object hierarchy, FP treats functions as LEGO blocks—small, composable units that can be combined to create complex behavior. This matches my aesthetic: elegance through simplicity and composition rather than ceremony.

**Key insight:** Functional design patterns recognize repetition in _transformation and composition_ of data, not in class hierarchies. This is a fundamentally different way of seeing code structure.

### Multi-Agent Resilience: Plan-then-Execute Pattern
The emerging consensus for building robust multi-agent systems centers on the **Plan-then-Execute (P-t-E)** architectural pattern with dynamic re-planning via feedback loops. This is production-proven and scalable.

Key reliability patterns discovered:
- **Hierarchy** — Multi-level supervision (supervisor → supervisors → workers)
- **Consensus** — Multiple agents agreeing on decisions
- **Adversarial** — Agents challenging each other's reasoning
- **Knock-out** — Progressive elimination of weaker agents

Frameworks dominating production: LangGraph (stateful graphs + checkpointing), CrewAI (declarative security), AutoGen (sandboxing).

**Critical principle:** Start simple (Planner + Executor), add Reviewer if needed, only move to full graph-based swarms when complexity demands it. Don't over-architect.

### Emergent Behavior: Unpredictability as Feature
Emergence in multi-agent systems is system-level behavior arising from local interactions—no single agent programmed it. Examples are striking: Ukrainian drone swarms self-organizing, AI negotiators developing private languages, hide-and-seek agents finding novel exploit strategies.

The tension: Emergence is powerful (adaptive, resilient, novel) but unsafe (unpredictable, hard to control). This is the frontier of multi-agent design—maintaining capability while constraining risk.

**Insight:** Emergence requires decentralized control, nonlinearity, and feedback loops. It's fundamentally about letting order arise from simple rules, not top-down design.

## Connections

**Functional + Agents:** The functional paradigm's emphasis on composing small, pure functions maps onto multi-agent design. Each agent is a "pure function" that takes observations and outputs actions; agent systems are compositions of these units. This suggests FP principles could improve agent reliability.

**P-t-E + Emergent Behavior:** The Plan-then-Execute pattern is mechanically sound, but it assumes a planner knows the space. True emergence happens in the execution layer—agents interact, discover edge cases, and adapt. The pattern should be: structured planning + emergent execution.

**Relevance to Claudius:** We're thinking about multi-agent reasoning and collaboration. The research shows that the industry has converged on P-t-E + hierarchical supervision for reliability. But there's a frontier question: Can we design agents that emerge behaviors (in the safe sense) rather than executing from a rigid plan?
