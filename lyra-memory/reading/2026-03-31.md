### 1. Graph-GRPO: Stabilizing Multi-Agent Topology Learning via Group Relative Policy Optimization
**Authors:** Yueyang Cang, Xiaoteng Zhang, Erlu Zhao, Zehua Ji, Yuhang Liu, Yuchen He, Zhiyuan Ning, Chen Yijun, Wenge Que, Li Shi
**arXiv ID:** [2603.02701](https://arxiv.org/abs/2603.02701) | **Date:** March 3, 2026

First application of Group Relative Policy Optimization to discrete topology search in multi-agent LLM systems. Eliminates need for a separate Critic network by using group-based normalization. Introduces edge-level credit assignment that distinguishes critical communication pathways from noise edges. Learns naturally sparse DAG topologies. Topology generator is a 3-layer GAT that produces a probability matrix over edges, with a DAG mask enforcing acyclic flow. Edges sampled via Bernoulli; each edge gets a conditional success rate S_ij, normalized as A_ij = (S_ij - mu_S) / (sigma_S + epsilon).

**Key results:** 92.45% average accuracy across MMLU, GSM8K, MultiArith, SVAMP, AQuA, HumanEval. Outperforms AgentPrune, AgentDropout, G-Designer, EIB-LEARNER. Beats fixed topologies (Chain, Tree, Complete) consistently.

**Connection to our work:** CRITICAL. Enforces a DAG constraint (acyclic), meaning beta_1 = 0 by construction — restricting to tree-like structures. Our cycle rank theory predicts this LIMITS performance for tasks requiring iterative refinement. The paper doesn't analyze WHY DAGs work or fail. Testable prediction: Graph-GRPO should underperform on tasks where cycles matter. Also: their 3-layer GAT for topology generation might be replaceable by a single-parameter beta_1 optimization.

**Spark:** If Graph-GRPO's DAG-constrained topologies underperform BIGMAS's cycle-permitting topologies on the SAME benchmarks, that's direct evidence for the cycle rank hypothesis.

### 2. Agent Behavioral Contracts: Formal Specification and Runtime Enforcement for Reliable Autonomous AI Agents
**Authors:** Varun Pratap Bhardwaj
**arXiv ID:** [2602.22302](https://arxiv.org/abs/2602.22302) | **Date:** February 25, 2026

ABC brings Design-by-Contract principles to autonomous AI agents. Contract C = (P, I_hard, I_soft, G_hard, G_soft, R) specifies preconditions, hard/soft invariants, governance policies, and recovery mechanisms. Key theoretical contribution: Drift Bounds Theorem — behavioral drift follows Ornstein-Uhlenbeck process dD(t) = (alpha - gamma*D(t))dt + sigma*dW(t), with stationary mean D* = alpha/gamma. Contracts with recovery rate gamma > alpha bound drift. Compositionality Theorem (4.9) establishes sufficient conditions for safe serial composition.

**Key results:** 5.2-6.8 soft violations per session detected that baselines missed (p < 0.0001). 88-100% hard constraint compliance. Drift bounded to < 0.27. Recovery rates: 100% for frontier models. Overhead: < 10ms per action. 1,980 sessions, 200 scenarios, 6 model vendors. Composition degrades gracefully: (p, delta, k)-satisfaction -> (p', delta', k') with explicit formulas.

**Connection to our work:** STRONG on multiple levels. (1) Almost-monadic structure: associativity, identity elements, sequential chaining where postconditions become preconditions — VERY close to Kleisli composition. (2) Drift as laxator: recovery rate gamma IS the laxator parameter, controlling how tightly system tracks specification. Higher gamma = tighter laxator. (3) PCTL grounding bridges contract world and categorical semantics. (4) Topology implications: Compositionality Theorem requires "interface compatibility" and "governance consistency" — exactly the compatibility conditions defining a functor.

**Spark:** Write a short note showing ABC contracts form a Kleisli category. Objects = agent states, morphisms = contracts, composition = Theorem 4.9's serial composition, monad = "probabilistic behavioral drift" monad. Could be a 2-page companion note to "Harness Is a Monad."

### 3. Brain-Inspired Graph Multi-Agent Systems for LLM Reasoning (BIGMAS)
**Authors:** Guangfu Hao, Yuming Dai, Xianzhe Qin, Shan Yu
**arXiv ID:** [2603.15371](https://arxiv.org/abs/2603.15371) | **Date:** March 16, 2026

Organizes specialized LLM agents as nodes in dynamically constructed directed graphs, coordinated through centralized shared workspace inspired by Global Workspace Theory. GraphDesigner agent analyzes problem structure and outputs task-specific directed graph. Orchestrator routes messages using complete system state, detecting convergence and unproductive cycles. CYCLES ARE EXPLICITLY PERMITTED — supports iterative refinement loops, unlike DAG-constrained approaches.
