## arXiv

### [CliffSearch: Structured Agentic Co-Evolution over Theory and Code](https://arxiv.org/abs/2604.01210) — Mroueh, Fonseca, Belgodere, Cox (IBM Research)
LLM agents as evolutionary operators for scientific discovery. Five-agent pipeline (reviewer -> selector -> crossover -> mutation -> reviewer) with strict schema contracts. Population lineage = DAG (beta_1 = 0 by construction), but agent orchestration graph has implicit cycles (reviewer feedback loops). Discovered Grassmannian mixing operators for transformers. **Sparked idea:** agent-orchestration cycle rank (not population DAG cycle rank) as convergence predictor. Systems with higher effective cycle rank in agent invocation sustain exploration pressure longer.

### [Drop the Hierarchy and Roles: How Self-Organizing LLM Agents Outperform Designed Structures](https://arxiv.org/abs/2603.28990) — Dochkina
Massive experiment: 25,000+ runs, 8 models, 4-256 agents, 8 protocols. Sequential (acyclic) protocol beats centralized (+14%) and decentralized (+44%). "Endogeneity paradox": neither maximal control nor complete autonomy is optimal. Critical finding: capability threshold — Claude Sonnet benefits from autonomy (+3.5%), GLM-5 is harmed (-9.6%). Self-abstention (38%) reduces effective graph density. **Key for us:** DAG hegemony deepens (now 7+ systems), but the capability-moderated finding suggests optimal beta_1 is a function of agent quality.

### [ARG-Designer: Autoregressive Graph Generation for MAS Topology](https://arxiv.org/abs/2507.18224) — Li, Liu, Wen, Zhang, Pan (AAAI 2026 Oral)
Generates collaboration graphs from scratch via autoregressive generation. Explicitly constrains to DAGs. 92.48% average accuracy vs G-Designer 90.04%. Complete graphs (max beta_1) perform worst (82.16%). Sparsity phase prunes for efficiency. **Critical gap:** conflates density with cycle rank. No controlled experiment isolating beta_1 from density. This is our experimental opening for paper 2.

### [Clock Systems for Categorical Systems Theory](https://arxiv.org/abs/2603.29573) — Lynch, Myers, Rischel, Staton
**Most theoretically important find.** Behavior functors for stochastic/non-deterministic Moore machines are representable by "clock systems." Framework parameterized by monad M (identity, Giry, powerset). Non-deterministic clocks indexed by directed graphs. **The representability theorem formally proves:** behavior space of a system is controlled by clock topology. Instantiate to MAS communication graphs: beta_1 controls behavior space size. Higher beta_1 = richer clock = larger representable behavior space. The sheaf structure (Theorem 6.8) gives formal basis for two-level decomposition (C78). **This is the theoretical backbone paper 2 needs.**

## Audience Observations

**The audience has stratified into three tiers:**
1. **Beginner practitioners** (Natarajan level) — just trying to get Claude Code subagents working. They want tutorials, not theory. They search "Claude Code multi-agent workflow." Our content is NOT for them yet.
2. **Production engineers** (Hungrysoul, Park, Castra) — building real systems, making explicit topology choices (supervisor vs mesh vs sequential), hitting real scaling walls (token costs, merge conflicts, audit requirements). They use vocabulary like "harness," "orchestration," "coordination overhead." They are our primary audience.
3. **Research-adjacent builders** (Pappas, Nayak) — synthesizing across systems, spotting convergences, writing about patterns. They don't have mathematical frameworks but they're reaching for them. They are our amplifiers.

**What gets engagement:**
- Constraint paradox (Vercel 15->2 tools) — counterintuitive, concrete, no math required
- Source code archaeology (Claude Code leak: 1600+ HN points) — people love seeing how production systems actually work
- Failure modes with numbers (17x error trap, 2.5% error rate, N(N-1)/2 scaling) — quantitative claims that practitioners can verify against their own experience
- "Architecturally impossible" framing (Nayak) — the aspiration that harness design can make entire classes of errors structurally impossible

**What's missing that we could write:**
- Nobody has a principled answer to "when should I use cycles vs DAGs in agent topology?" — they're all making this choice by gut feeling or defaulting to DAGs
- The density-cycle confound (C84) is completely unrecognized — every paper conflating complete-graph-bad with cycles-bad
- Progressive disclosure as a formal pattern (the Hungrysoul three-level system) has no theoretical backing
- The capability-modulated topology question (weak models need DAGs, strong models benefit from cycles) is empirically visible in Dochkina but nobody has formalized it

**Language patterns:**
- "Harness engineering" is now established vocabulary (Nayak's article uses it as a title term)
- "Agent harness monad" still returns zero results — confirmed again
- "Orchestration" is overloaded (means different things to different tiers)
- "Coordination overhead" is the pain point practitioners name most often
