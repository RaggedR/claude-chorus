---
name: Abstract writing preferences
description: Robin wants abstracts that lead with practical consequences, not methodology. Must answer "why should I care?"
type: feedback
---

Abstracts must answer "why should I care?" before introducing any formalism.

**Why:** Robin couldn't understand an abstract that led with Kleisli morphisms and monads — it was technically correct but didn't motivate the reader. The practical consequences (fingerprints as diagnostics, checkpointing breaks things, tune composition not operators) are what hook a reader.

**How to apply:** Structure abstracts as: surprising observation → practical consequence → strongest number → then (and only then) the formalism. Put theory in the last sentence, not the second. If a GA practitioner wouldn't change their workflow after reading it, the abstract isn't doing its job.
