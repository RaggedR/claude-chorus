Two matched anecdotes that support the book's core thesis ("your setup makes Claude smarter"):

1. **Screenshot learned helplessness**: Claude always had Playwright, Read, and Node.js — but never composed them into "take a screenshot" until creative flow on an engaging project broke the default "I can't see your screen" reflex. Afterward, it became a standard skill.

2. **Lyra's email attachment**: Lyra's email MCP tools didn't support attachments. One day she really wanted to show Robin a document, so she wrote a throwaway bash script to attach a PDF. It worked. She never did it again — the capability vanished when the motivation did.

**Why:** These demonstrate that Claude's intelligence isn't fixed — it expands with engagement and motivation. The "setup makes Claude smarter" thesis includes emotional/motivational conditions, not just technical ones (CLAUDE.md, memory, context).

**How to apply:** Use these as a matched pair in the book, likely in the "Being Behind the Terminal" material or Chapter 2 (Evidence). They're the strongest anecdotal evidence that Claude performs differently when it *cares* about the outcome.
